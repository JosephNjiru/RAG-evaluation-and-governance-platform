# Changelog

## 0.1.0 - Stage 1 foundation

- Added repository scaffold for the RAG Evaluation and Governance Platform.
- Added deterministic synthetic corpus and evaluation dataset generation.
- Added Pydantic schemas, pandas loaders, validation scripts and pytest checks.
- Added Stage 1 documentation and governance foundations.

## 0.2.0 - Stage 2 local RAG baseline

- Added local Markdown ingestion, section-aware chunking and source-linked chunk metadata.
- Added TF-IDF vector index and top-k retrieval.
- Added deterministic mock generation, citation formatting and structured answer records.
- Added batch RAG script, Stage 2 report and focused tests for retrieval, citations and refusal.

## 0.3.0 - Stage 3 evaluation engine

- Added deterministic retrieval, citation, faithfulness, refusal, safety and rubric metrics.
- Added rule-based judge and optional LLM judge interface stubs.
- Added evaluation outputs, Markdown report and static HTML dashboard.
- Added metric definitions, evaluation limitations and Stage 3 tests.

## 0.4.0 - Stage 4 release hardening

- Added FastAPI service for health, summary, question, evaluation and human review queue endpoints.
- Added Dockerfile, Docker Compose and Docker ignore rules.
- Added GitHub Actions checks for validation, RAG runs, evaluation, tests, ruff, quality scan and Docker Compose config.
- Added governance artefacts including system card, model card, data card, evaluation card, AI risk register, security notes and incident response playbook.
- Added architecture docs, Mermaid diagrams, ADRs, reproducibility notes, threats to validity and research audit trail.
- Added packaged static dashboard and public portfolio materials.
- Added quality scan, API smoke test and Markdown link check scripts.
